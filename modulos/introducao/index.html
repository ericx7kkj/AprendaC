Introdução ao C

Entenda o que é a linguagem C, como ela funciona, sua história e escreva seus primeiros programas do zero.

O que é a linguagem C?

C é uma das linguagens de programação mais antigas, influentes e ainda relevantes do mundo. Criada em 1972 por Dennis Ritchie nos Laboratórios Bell (AT&T), ela nasceu com um objetivo muito prático: reescrever o sistema operacional Unix de uma forma portável e eficiente.

Antes de C, sistemas operacionais eram escritos em Assembly — uma linguagem extremamente próxima do hardware, difícil e específica para cada processador. C trouxe o melhor dos dois mundos: a velocidade próxima do Assembly com uma sintaxe muito mais legível e portável.

💡 O kernel do Linux, que roda em bilhões de dispositivos Android, servidores e até na Estação Espacial Internacional, é escrito principalmente em C.

Por que aprender C em 2026?

  • Base de tudo: C++, Java, C#, Python e Go têm raízes diretas em C. Quem aprende C aprende o fundamento.
  • Entender o computador de verdade: C trabalha diretamente com memória, ponteiros e hardware — você realmente entende como as coisas funcionam.
  • Alta performance: sistemas embarcados, drivers, jogos e bancos de dados usam C justamente pela velocidade.
  • Mercado sólido: IoT, firmware, segurança ofensiva e sistemas operacionais ainda dependem fortemente de C.

Estrutura de um programa em C

Todo programa C tem uma estrutura mínima obrigatória. Veja e entenda cada parte:

hello.cprograma mínimo
#include <stdio.h>   // 1. inclui a biblioteca de I/O

int main() {             // 2. função principal — entrada do programa
    printf("Hello!\n");  // 3. imprime na tela
    return 0;            // 4. sinaliza sucesso ao sistema operacional
}
  • #include <stdio.h> — diretiva que importa a biblioteca padrão de entrada/saída. Sem ela, printf não existe.
  • int main() — ponto de entrada obrigatório. O sistema operacional sempre começa aqui. O int indica que main retorna um inteiro.
  • printf("Hello!\n") — imprime texto. O \n é uma sequência de escape que representa uma nova linha.
  • return 0; — devolve 0 ao SO, indicando que o programa terminou sem erros.

Sequências de escape

Dentro de strings, certos caracteres especiais são representados com barra invertida:

SequênciaSignificadoExemplo de uso
\nNova linhaprintf("Linha1\nLinha2");
\tTabulação (tab)printf("Nome:\tAna");
\\Barra invertida literalprintf("C:\\pasta");
\"Aspas dentro da stringprintf("Ele disse \"oi\"");
\0Caractere nulo (fim de string)Usado internamente

Comentários em C

Comentários são ignorados pelo compilador — servem para documentar e explicar o código:

comentarios.cdois estilos
// Comentário de linha — tudo após // é ignorado pelo compilador

/* Comentário de bloco — pode ocupar
   várias linhas, útil para
   documentação longa */

int main() {
    printf("Ola!\n");  // comentário ao lado do código
    return 0;
}

Erros mais comuns de iniciantes

ErroCausaSolução
Esquecer o ;C exige ; no final de toda instruçãoAdicionar ; antes de fechar o bloco
Não fechar { }Todo bloco aberto precisa ser fechadoVerificar se cada { tem seu }
Esquecer #includeFunções como printf não são reconhecidasAdicionar #include <stdio.h> no topo
Maiúsculas erradasC é case-sensitive: Printf ≠ printfUsar sempre letras minúsculas nas funções padrão
⚠️ Bom hábito: compile e execute o código frequentemente, em pequenos incrementos. Assim, quando um erro aparecer, você sabe exatamente onde está.
